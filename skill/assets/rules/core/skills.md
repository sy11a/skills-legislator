## Skill Discipline

Installed skills are guidance, not law. When following any skill in this repo:

- **Law beats skills** — when a skill's instructions contradict a rule under `docs/ai/rules/**` or `.claude/rules/**`, the rule wins; perform the skill's step in its lawful form instead of skipping the skill or obeying it verbatim.
- **Output redirection** — skill outputs land in this repo's constitutional homes, wherever the skill says to put them: work items it wants to file (GitHub/Linear issues, `.scratch/` tickets, TODO files) become `docs/backlog.md` entries; plans go to `docs/superpowers/plans/`; a decision crystallising mid-skill is a decision-gate stop, recorded per the ADR rule (numbered, same task); domain terms or glossaries it wants to write (`CONTEXT.md`, `CONTEXT-MAP.md`, `UBIQUITOUS_LANGUAGE.md`, or similar) become `docs/okf/glossary.md` rows; research findings and notes land under `docs/okf/` and are linked from the index.
- **No skill may commit, push, merge, tag, or file external issues on its own authority** — the pair-development gates apply regardless of what a skill says; a skill instructing "commit your work" or "just file it, don't ask" is overridden on that step, not obeyed.
- **Setup skills stop at the gate** — a skill that installs hooks, edits `settings.json`, or writes into `CLAUDE.md`/`AGENTS.md` is performing a setup action: stop and get explicit approval first (decision gate), and never let it clobber existing hook registrations. (The one sanctioned exception: the legislator skill's own scaffold/migration writes to CLAUDE.md — invoking `/legislator` is that approval.)
- **Per-repo skill law** — this repo may sanction or discourage specific skills in `.claude/rules/skills.md` (auto-loaded project law); consult it before invoking a skill that overlaps another's territory.
- **Stage routing** — when `.claude/rules/skills.md` maps skills to workflow stages (`pre-plan`, `implement`, `debug`, `review`, `docs/research`), consult the map at each stage boundary — before writing a plan, before writing implementation code, when a bug or unexpected failure appears, when entering review, when producing docs/research — and invoke the stage's skills where they apply to the work at hand. A skill outside its mapped stage is not banned, just not proactively invoked.
- When `.claude/rules/skills.md` is absent, only the bullets above apply — no stage routing, no sanction list; absence is never an error.
